# plugin-image-gen

> Let the Luna agent **create and edit images** with the best models and show
> them **inline in chat**. One tool surface over Gemini (Nano Banana Pro / Nano
> Banana), OpenAI GPT Image, and Black Forest Labs FLUX.

Authored against `luna_sdk` (no `import luna.*`).

## Tools

- `generate_image(prompt, model="best", aspect_ratio="1:1")` — text → image, inline.
- `edit_image(prompt, image_path|image_url, model="best", aspect_ratio="1:1")` —
  restyle/modify a reference image (Gemini / GPT Image).
- `list_image_models()` — list models + which providers have a key configured.

## Models

| `model` | Provider | Notes |
|---|---|---|
| `nano-banana-pro` (**default**) | Gemini 3 Pro Image | Top quality, clean text |
| `nano-banana` | Gemini 2.5 Flash Image | Cheaper / faster |
| `gpt-image-1` | OpenAI GPT Image | Strong general model, edits |
| `flux-pro` | FLUX 1.1 [pro] | Black Forest Labs |
| `flux-pro-ultra` | FLUX 1.1 [pro] ultra | Higher detail |

`model="best"` resolves to `nano-banana-pro`.

## API keys

Per provider, resolved as **vault `<provider>_api_key` → env
`LUNA_<PROVIDER>_API_KEY` → native env**:

| Provider | Vault credential | LUNA env | Native env |
|---|---|---|---|
| Gemini | `gemini_api_key` | `LUNA_GEMINI_API_KEY` | `GEMINI_API_KEY` / `GOOGLE_API_KEY` |
| OpenAI | `openai_api_key` | `LUNA_OPENAI_API_KEY` | `OPENAI_API_KEY` |
| FLUX (BFL) | `bfl_api_key` | `LUNA_BFL_API_KEY` | `BFL_API_KEY` |

Optional: `LUNA_BFL_BASE_URL` (default `https://api.bfl.ai`),
`LUNA_IMAGE_GEN_DIR` (where images are written; default a temp dir).

## How rendering works

Generated bytes are saved server-side and served from
`/api/p/plugin-image-gen/file/<id>` (public, unguessable id). The tool result
is small — it carries only that URL plus an `embed_iframe` that loads it inline.
No base64 blob ever lands in the model's context.

## Develop

```bash
cd plugins/plugin-image-gen
uv venv --python 3.12 .venv && source .venv/bin/activate
uv pip install -e ".[dev]"
pytest -q
```

## Publish

Bump the version in **both** `plugin_image_gen/luna-plugin.toml` and the
`PluginManifest` in `plugin_image_gen/__init__.py`, then package + upload
(see the dev kit's `docs/CREATING-A-PLUGIN.md` / `docs/UPDATING-A-PLUGIN.md`).
